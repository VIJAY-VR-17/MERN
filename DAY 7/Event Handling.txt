Event Handling

What is an Event?

An Event is an action or occurrence that happens in the browser. Events can be triggered by the user (clicking a button, typing on the keyboard) or by the browser (page loading, resizing).

Examples of events:

Clicking a button
Pressing a key
Moving the mouse
Submitting a form
Loading a webpage
What is Event Handling?

Event Handling is the process of writing JavaScript code that responds when an event occurs.

Syntax
element.addEventListener("event", function);
element → HTML element
event → Type of event (click, keyup, mouseover, etc.)
function → Code to execute
Ways to Handle Events
1. Inline Event
<button onclick="showMessage()">Click</button>

<script>
function showMessage(){
    alert("Button Clicked");
}
</script>

Output:

Button Clicked
2. Using JavaScript Property
<button id="btn">Click</button>
let button = document.getElementById("btn");

button.onclick = function(){
    alert("Button Clicked");
};
3. Using addEventListener() (Recommended)
<button id="btn">Click</button>
let button = document.getElementById("btn");

button.addEventListener("click", function(){

    alert("Button Clicked");

});

Output:

Button Clicked

Form Events
Event	Description
submit	Form submitted
change	Input value changes
input	Fires whenever value changes
focus	Input gets focus
blur	Input loses focus

Example

form.addEventListener("submit", function(event){

    event.preventDefault();

    alert("Form Submitted");

});
Event Object

Whenever an event occurs, JavaScript creates an Event Object.

button.addEventListener("click", function(event){

    console.log(event);

});

Useful Properties

event.target

Returns the clicked element.

event.type

Returns the event name.

Output

click
event.preventDefault();

Stops the browser's default behavior.

Example

Without preventDefault()

Form reloads page.

With preventDefault()

Page does not reload.
Event Bubbling

When an event occurs, it first happens on the target element and then moves upward through its parent elements.

Example

2. Event Delegation
What is Event Delegation?

Instead of attaching event listeners to every child element, attach one event listener to the parent.

It works because of Event Bubbling.

Without Event Delegation

item1.onclick=function(){}

item2.onclick=function(){}

item3.onclick=function(){}

Three listeners are created.

With Event Delegation

<ul id="list">

<li>HTML</li>

<li>CSS</li>

<li>JavaScript</li>

</ul>
let list=document.getElementById("list");

list.addEventListener("click",function(event){

    console.log(event.target.textContent);

});

Output

Click HTML

HTML

Click CSS

CSS

Click JavaScript

JavaScript

Advantages

Less memory usage
Better performance
Works for dynamically created elements
Less code

5. Local Storage
What is Local Storage?

Local Storage stores data permanently inside the browser.

Data remains even after closing the browser.
Stores only strings.
Capacity about 5–10 MB (varies by browser).
Store Data
localStorage.setItem("name","Rahul");
Retrieve Data
console.log(localStorage.getItem("name"));

Output

Rahul
Remove Item
localStorage.removeItem("name");
Clear All Data
localStorage.clear();
Example
localStorage.setItem("city","Chennai");

console.log(localStorage.getItem("city"));

Output

Chennai
6. Session Storage
What is Session Storage?

Session Storage stores data only for the current browser session.

Once the tab is closed, all stored data is removed.

Store Data
sessionStorage.setItem("user","Rahul");
Retrieve Data
console.log(sessionStorage.getItem("user"));

Output

Rahul
Remove Item
sessionStorage.removeItem("user");
Clear Storage
sessionStorage.clear();
